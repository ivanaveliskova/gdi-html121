Semantic HTML

In order for the internet to be accessible to anyone and everyone, your elements should be as meaningful as possible.

Poor example:

<p>This is a heading</p>
<h2>This is a lot of text and really poor use of an h2 and stuff</h2>

This is a heading

This is a lot of text and really poor use of an h2 and stuff

Although there are limitations to what semantic markup we have available to us, we can try to make it as semantic as possible with what we have.

Semantic HTML

  • If you need to write a bunch of text a <p> tag is appropriate
  • For the same reason, something important and short should use header tags (h1, h2, h3, etc.)

    Don't style your <p> tags to look like <h1> tags!!

  • In order for your page to be valid, you can only use one (1) <h1> tag.
  • Tables are for tabular data!

    You wouldn't use a spreadsheet for your resume, so don't use a table for page layout.

You will learn about new meaningful/semantic elements in the HTML 201 class, here we will work with what we already have.

Tables: You wouldn’t use a spreadsheet to write out something like your resume, just like you wouldn’t use a word document to track tabular data such as the ratio of yellow to brown to green to red to blue m&ms in each packet (fun fact, I actually did this in college, never did anything with the data, but wanted to).

We can use CSS to assist in making our pages remain responsive.

  • Percentage widths
    • Also percentage margins
    • You can also use calc()
  • Percentage widths: we can set an element to be a certain width relative to it's container
    • Using percentage margins will allow us create a page that has a full 100% so things stay aligned at all different sizes.
    • Note about using percentage margins/padding: see next slide!
    • You can use pixels and then use calc figuring out the remaining width from 100% → but we need to remember the box model for this and how margins are calculated on an element and surrounding elements <-- limited browser support

MOAR CSS!

  • min- and max-width

Use percentage widths and min/max-width in combination for some cRaZy stuff

                            img {
    width: 50%;       /* Images will be 50% of their container */
    max-width: 250px; /* Until the image hits 250px
                         then it will remain at 250px */
    min-width: 100px; /* If the image starts to shrink
                         and suddenly 50% is not legible,
                         or doesn't look good,
                         this is as small as it will get */
}
  • My suggestion of using min-width is for images. Or for elements that you don't want trying to squeeze into a small space (maybe a side nav) example: image on my site about me page
  • Max-width will keep your elements from getting too big, or stretching out too much and suddenly there being all this ridiculous white space everywhere on ginormous screens

And by crazy, I mean nice clean responsive images/sections

Fonts

Fonts can be "responsive," too!

We can use font size known as ems.

What is an em?

It is a measure of font size originally determined by the width of the letter 'M'

You can set a base font on the body of your document such as 16px. And then you don't have to touch pixels again. You can use ems based off of that base font size.

We all can't do the math in our heads to figure out what ems we want so here's a handy link!

Ems are determined by the width of the letter 'M.' That size varies based on the base font size. (16px sized em is different from 10px sized em). So ems work a bit like percentages but are geared more towards use in fonts

CSS

Fonts

Example

                            body {
    font-size: 30px; /* This is our base font */
}

div {
    font-size: 0.75em; /* All divs will have 22px font */
}

h1 {
    font-size: 1.167em; /* All h1s will have 35px font */
}

This is great because if we later decide to change our minds about our font size and want to use a larger font we can easily just change the base font once and everything else will fix itself accordingly.

CSS

Fonts

There is a solution!

                            body {
    font-size: 30px; /* This is our base font */
}

div {
    font-size: 0.75em; /* All divs will have 22px font */
}

div div {
    font-size: 1em; /* Sets any divs within divs to have font equal to the last time font-size was defined */
}
This is the base font size.
This is ok and no problem.
Yay! This is fixed!

As a side note: you probably don't want to be setting your ems to divs because that with the rules of cascading, that can lead you to all kinds of trouble with putting paragraphs within divs etc.
